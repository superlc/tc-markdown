# Proposal: optimize-strikethrough-streaming

## Why
当前流式渲染删除线（GFM `~~`）时，会出现“先以普通文本显示，随后又突然变成删除线”的闪现。
这类样式跃迁会显著破坏流式阅读体验，尤其在 chunk 边界落在 `~~` 或删除线内容附近时更明显。

## What Changes
- 明确删除线在流式场景下的期望行为：当 `~~` 已确定为删除线开始时，后续内容应尽可能以删除线形态逐字呈现，而不是先以纯文本呈现再回溯转换。
- 补充/收紧 `markdown-streaming` spec 中“删除线预测”的场景描述，覆盖常见 chunk 边界情况与“无闪现”要求。
- 不改变 Markdown 语义解析结果；仅优化“未闭合/边界不完整”时的显示策略，目标是减少视觉闪烁。

## Non-Goals
- 不新增新的 Markdown 语法。
- 不重写 remark/rehype 解析链路。
- 不在本变更中引入复杂的动画过渡（仅处理闪现根因）。
